


# 프로세스 관리

## 1. 프로세스 확인 (ps, htop)

### ps

``` bash
ps [옵션]
```

-   `ps` (process state)
-   현재 실행 중인 프로세스 상태 출력

### 주요 옵션

  옵션          설명
  ------------- -------------------------------
  -A / -e       모든 프로세스 출력
  -a            터미널과 연관된 프로세스 출력
  -f            풀 포맷 출력
  -l            상세 정보 출력
  -u \[user\]   특정 사용자 프로세스
  -p \[pid\]    특정 PID 조회

### 실행 예시

``` bash
$ ps -f
UID       PID    PPID  TTY      STIME     COMMAND
user     2013    1793  pts/0    14:10     ps -f
user     1793    1792  pts/0    13:31     bash
user     1792       1  ?        13:31     mintty
```

------------------------------------------------------------------------

### htop

-   터미널 기반 프로세스 모니터링 도구
-   실시간 CPU / 메모리 사용량 확인 가능

``` bash
htop
```

------------------------------------------------------------------------

## 2. 포그라운드 / 백그라운드

### 개념

-   포그라운드: 사용자와 상호작용
-   백그라운드: 사용자 입력 없이 실행
-   Stopped: 일시 정지 상태

### 명령어

``` bash
jobs
fg
bg
```

### 예시

> 두 개의 터미널을 사용

#### Terminal 1

``` bash
$ sleep 100
```

#### Terminal 2

``` bash
$ jobs
$ ps
```

------------------------------------------------------------------------

## 3. 프로세스 종료 (kill, pkill)

``` bash
kill -[signal] [PID]
```

  번호   이름      설명
  ------ --------- -----------
  2      SIGINT    Ctrl+C
  9      SIGKILL   강제 종료
  15     SIGTERM   정상 종료

``` bash
kill -9 [PID]
```

``` bash
pkill [이름]
```

------------------------------------------------------------------------

## 4. 시그널

``` bash
kill -l
```

------------------------------------------------------------------------

## 5. 우선순위 (nice)

``` bash
nice -n [값] [명령어]
renice [값] [PID]
```

-   범위: -20 \~ 19
-   낮을수록 우선순위 높음

------------------------------------------------------------------------

## 6. 핵심 정리

-   ps / htop: 프로세스 확인
-   jobs / fg / bg: 제어
-   kill / pkill: 종료
-   nice / renice: 우선순위

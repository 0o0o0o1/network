# 프로세스 관리

## 1. 프로세스 확인 (ps, htop)

### ps

ps는 **process status**의 약자로 현재 실행 중인 프로세스 상태를 출력한다.

```bash
ps [옵션]
```

### 주요 옵션

| 옵션 | 설명 |
|------|------|
| -e | 모든 프로세스 출력 |
| -a | 터미널과 연관된 프로세스 출력 |
| -f | 풀 포맷 출력 |
| -l | 상세 정보 출력 |
| -u [user] | 특정 사용자 프로세스 |
| -p [pid] | 특정 PID 조회 |

### 실행 예시

```bash
$ ps
```

- 현재 터미널에서 실행 중인 프로세스 확인

```bash
$ ps -ef
```

- 전체 프로세스를 상세 정보와 함께 출력

```bash
UID       PID    PPID  C STIME TTY      TIME     CMD
user     2013    1793  0 14:10 pts/0    00:00:00 ps -ef
user     1793    1792  0 13:31 pts/0    00:00:00 bash
```

### htop

htop은 터미널 기반 프로세스 모니터링 도구로, 실시간으로 시스템 상태를 확인할 수 있다.

```bash
$ htop
```

- CPU, 메모리, 프로세스를 실시간으로 확인 가능
- 인터랙티브 UI 제공

## 2. 포그라운드 / 백그라운드

### 개념

- 포그라운드 프로세스: 사용자와 직접 상호작용하며 실행
- 백그라운드 프로세스: 사용자 입력 없이 실행
- Stopped 상태: 실행이 일시 중지된 상태

### 실행 흐름

#### 포그라운드 실행

```bash
$ sleep 100
```

- 터미널을 점유하며 실행됨

#### 백그라운드 실행

```bash
$ sleep 100 &
```

- 터미널을 점유하지 않고 실행

### 작업 확인

```bash
$ jobs
[1]+  Running   sleep 100 &
```

- 현재 백그라운드 작업 목록 확인

### 포그라운드 전환

```bash
$ fg %1
```

### 백그라운드 전환

```bash
$ bg %1
```

## 3. 프로세스 종료 (kill, pkill)

### kill

kill은 특정 프로세스를 종료할 때 사용한다.

```bash
kill -[signal] [PID]
```

### 실행 흐름

#### PID 확인

```bash
$ ps -ef | grep sleep
```

#### 정상 종료

```bash
$ kill 1234
```

#### 강제 종료

```bash
$ kill -9 1234
```

### pkill

```bash
$ pkill sleep
```

---

## 4. 시그널 (Signal)

시그널은 커널이 프로세스에 전달하는 제어 신호이다.

```bash
$ kill -l
```

### 주요 시그널

| 번호 | 이름 | 설명 |
|------|------|------|
| 2 | SIGINT | Ctrl + C |
| 9 | SIGKILL | 강제 종료 |
| 15 | SIGTERM | 정상 종료 |

---

## 5. 프로세스 우선순위 (nice)

### 개념

nice 값은 프로세스의 우선순위를 결정한다.

- 범위: -20 ~ 19
- 값이 낮을수록 우선순위 높음
- 기본값: 0

### 실행 시 설정

```bash
$ nice -n 10 sleep 100
```

### 실행 중 변경

```bash
$ renice 10 1234
```

---

## 6. 핵심 정리

- ps / htop: 프로세스 확인
- jobs / fg / bg: 실행 상태 제어
- kill / pkill: 프로세스 종료
- signal: 프로세스 제어 신호
- nice / renice: 우선순위 관리

# 프로세스 개념

## 1. 프로세스란?

프로세스(Process)는 **컴퓨터에서 실행 중인 프로그램**을 의미한다.

사용자 또는 이미 실행 중인 프로세스가 실행 가능한 프로그램을 실행하고자 할 때,  
운영체제에 실행을 요청하고 운영체제는 이를 받아 새로운 프로세스를 생성한다.

생성된 프로세스는 CPU에서 실행되며, 여러 프로세스가 동시에 존재하는 경우  
운영체제가 스케줄링을 통해 실행 순서를 조절한다.

---

## 2. 프로세스 상태

프로세스는 다음과 같은 5가지 상태를 가진다.

| 상태 | 설명 |
|------|------|
| Create | 프로세스가 생성되는 중 |
| Running | CPU를 점유하여 명령어를 실행 중 |
| Ready | CPU 할당을 기다리는 상태 |
| Blocked | 입출력 대기 등으로 실행 불가능한 상태 |
| Terminated | 실행이 종료된 상태 |

---

## 3. PID와 PPID

리눅스의 모든 프로세스는 고유한 식별자를 가진다.

- **PID (Process ID)** : 현재 프로세스의 고유 ID  
- **PPID (Parent Process ID)** : 부모 프로세스의 ID  

PID는 시스템 내에서 중복되지 않는다.

---

## 4. 프로세스 생성

리눅스에서는 새로운 프로세스를 생성할 때 `fork()`를 사용한다.

- `fork()` : 현재 프로세스를 복제하여 새로운 프로세스 생성
- 부모 프로세스(Parent Process) → 자식 프로세스(Child Process) 생성

이후 자식 프로세스에서 `exec()`를 사용하면  
다른 프로그램으로 덮어써서 새로운 프로그램을 실행할 수 있다.

---

## 5. 프로세스 구조

리눅스 프로세스는 트리 구조를 가진다.

- 최상위 프로세스는 **init 프로세스**
- init 프로세스의 PID는 **1**

모든 프로세스는 부모-자식 관계를 가진다.

---

## 6. 특수 프로세스

### 고아 프로세스 (Orphan Process)
부모 프로세스가 먼저 종료된 경우

- PPID가 1(init)으로 변경됨
- init 프로세스가 대신 관리

---

### 좀비 프로세스 (Zombie Process)
자식 프로세스가 종료되었지만, 부모가 종료 처리를 하지 않은 상태

- 시스템 자원을 일부 점유
- 부모 프로세스가 `wait()` 호출 시 정리됨

---

## 7. 핵심 정리

- 프로세스는 실행 중인 프로그램이다.
- 운영체제가 프로세스 생성 및 실행을 관리한다.
- `fork()`와 `exec()`를 통해 새로운 프로그램이 실행된다.
- 모든 프로세스는 PID와 PPID를 가진다.
- 리눅스는 트리 구조의 프로세스 구조를 가진다.
- 고아 프로세스와 좀비 프로세스는 특수한 상태이다.

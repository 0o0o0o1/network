# 파일 디스크립터

## 1. 파일 디스크립터란?

파일 디스크립터(File Descriptor, fd)는\
프로세스가 파일에 접근하기 위해 사용하는 **정수 형태의 식별자**이다.

프로세스가 파일을 열어 작업을 수행할 때,\
운영체제는 해당 파일을 관리하기 위한 권한을 부여하고\
파일 디스크립터(fd)를 할당한다.

프로세스는 이 fd를 이용하여 파일에 접근한다.

------------------------------------------------------------------------

## 2. 동작 구조

프로그램이 실행되면 파일 디스크립터를 관리하는 **File Descriptor
Table**이 생성된다.

이 구조는 다음과 같이 연결된다.

File Descriptor Table → Open File Description → inode

-   File Descriptor Table: fd를 관리하는 테이블\
-   Open File Description: 열린 파일에 대한 정보\
-   inode: 실제 파일 정보 (디스크 상 위치 등)

------------------------------------------------------------------------

## 3. 파일 접근 방식

프로세스는 파일 디스크립터 테이블의 **인덱스 번호(fd)**를 이용해 파일에
접근한다.

fd → 파일

즉, fd는 파일을 직접 가리키는 것이 아니라\
**테이블을 통해 간접적으로 접근**한다.

------------------------------------------------------------------------

## 4. 기본 파일 디스크립터

프로세스는 실행될 때 기본적으로 다음 fd를 할당받는다.

  fd   이름     설명
  ---- -------- ----------------
  0    stdin    표준 입력
  1    stdout   표준 출력
  2    stderr   표준 에러 출력

이들은 프로그램 시작 시 운영체제가 자동으로 할당한다.

------------------------------------------------------------------------

## 5. 추가 파일 디스크립터

프로세스가 `open()` 등의 시스템 호출로 새로운 파일을 열면\
파일 디스크립터는 **3번부터 순차적으로 할당**된다.

0 → stdin\
1 → stdout\
2 → stderr\
3 → 새로 열린 파일

------------------------------------------------------------------------

## 6. 핵심 정리

-   파일 디스크립터는 파일 접근을 위한 정수 값이다.\
-   프로세스는 fd를 통해 파일에 접근한다.\
-   fd는 테이블을 통해 inode까지 연결된다.\
-   기본 fd는 0, 1, 2이며, 이후는 3부터 할당된다.

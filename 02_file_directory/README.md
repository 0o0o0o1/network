# 파일 및 디렉토리

## 1. 파일 (File)
파일은 의미 있는 정보를 모아둔 **논리적인 데이터 단위**로, 종이 문서의 묶음을 의미하는 ‘파일(File)’에서 유래한 용어이다.

- 목적에 따라 다양한 형태의 파일이 존재한다.
- 일반적으로 파일 이름은 `<파일이름>.<확장자>` 형태로 구성된다.

### 파일 식별 방식
GUI 환경에서는 확장자를 기준으로 파일을 구분하지만,  
CLI 환경에서는 운영체제가 파일의 **앞부분 일부 바이트를 읽어 파일 유형을 판단**한다.

이때 파일의 맨 앞에 위치하여 파일의 종류를 나타내는 값을 다음과 같이 부른다.

- Magic Number
- File Signature

---

## 2. 디렉터리 (Directory)
디렉터리는 파일을 체계적으로 관리하기 위한 **네임스페이스(Namespace)**이며, 일반적으로 ‘폴더’라고 불린다.

- 디렉터리는 파일 또는 다른 디렉터리를 포함할 수 있다.
- 리눅스에서는 디렉터리를 **특수한 파일**로 취급하여 관리한다.

### 디렉터리 구조
- **하위 디렉터리 (Sub Directory / Child Directory)**  
  → 특정 디렉터리 내부에 포함된 디렉터리

- **상위 디렉터리 (Parent Directory)**  
  → 하위 디렉터리를 포함하는 디렉터리

---

## 3. 리눅스 주요 디렉터리

```bash
/bin    # 기본 명령어
/boot   # 부팅 관련 파일
/dev    # 디바이스 파일
/etc    # 설정 파일
/home   # 사용자 홈 디렉터리
/lib    # 라이브러리
/lost+found
/media  # 외부 장치 마운트
/mnt    # 임시 마운트
/opt    # 추가 소프트웨어
/proc   # 프로세스 정보
/root   # 관리자 홈
/sbin   # 관리자 명령어
/srv    # 서비스 데이터
/tmp    # 임시 파일
/usr    # 사용자 프로그램
/var    # 로그 및 가변 데이터
```

---

## 4. 경로 (Path)

### 절대 경로 (Absolute Path)
- 최상위 디렉터리 `/`부터 시작하는 경로

```bash
/home/user/file.txt
```

---

### 상대 경로 (Relative Path)
- 현재 위치를 기준으로 한 경로

```bash
./file.txt     # 현재 디렉터리
../file.txt    # 상위 디렉터리
~/file.txt     # 홈 디렉터리
```

---

## 5. 핵심 정리
- 파일은 데이터를 저장하는 논리적인 단위이다.
- 디렉터리는 파일을 관리하는 구조이다.
- 리눅스에서는 디렉터리도 파일의 일종이다.
- 파일은 Magic Number로 식별될 수 있다.
- 파일 시스템은 트리 구조로 구성된다.
- 경로는 절대 경로와 상대 경로로 나뉜다.
